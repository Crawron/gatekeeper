RenderResult

RenderResult: ReplyComponent | string | number | boolean | undefined | null | RenderResult[]
  • Defined in core/reply-component.ts:58

Anything that can be rendered in a reply.

  • Embed components, action row components, and text components are accepted as-is
  • Button and select menu components are automatically placed in action rows, respecting discord's restrictions
  • Strings and numbers become text components. Empty strings can be used to add empty lines in the message. \n will also add a new line.
  • Nested arrays are flattened
  • Everything else (booleans, undefined, null) is ignored

UseClientOptions

UseClientOptions: { useGlobalCommands?: boolean; useGuildCommands?: boolean }
  • Defined in core/gatekeeper.ts:49

Options for attaching a client to a gatekeeper instance.

Type declaration

  • Optional useGlobalCommands?: boolean

    Register global commands.

    Global commands can be used from any server, and take a while to show up, so this isn't great for testing. I'd recommend only enabling this if you're scaling the bot up to many servers/channels, when using guild commands reaches the 100 total command limit for a bot.

    default

    false

  • Optional useGuildCommands?: boolean

    Registers commands per guild.

    The commands are immediately available for use, which makes this much better for development, and works fine for small bots that are only used in a few servers.

    default

    true
